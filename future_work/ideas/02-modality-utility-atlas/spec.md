# Idea 02 — Modality-Utility Atlas:模态证据效用地图与预检工具包

- 目标会议:RecSys 2027(长文或资源轨)| 层级:A | 纯离线,无 GPU 竞争
- 一句话:在 20+ 数据集 × 3+ 编码器 × {文本,图像,图文} 上系统测绘 U_ds 分布,发布"负采样安全度"地图与预检工具,把 AAAI'27 的 4 点反转升级为领域级测量事实。

## 1. Gap

1. 多模态推荐论文选数据集靠惯例(Beauty/Clothing/Sports 三件套),**没有任何工作测量过"该数据集的模态证据到底预测正例到什么程度"** — 而我们已证明这个量决定内容信号作为负例源的安全性;
2. 基准层面:Binge Watch 刚发布 M3L-10M/20M(2602.15505)、MMPCBench(2601.19750)在补内容,但都不刻画**证据效用几何**;SIDInspector(2606.10375)诊断 tokenizer 而非原始证据;
3. 方法论层面:我们的零点校准教训(原始一致性地板 0.93-0.95 随几何漂移)说明**跨数据集比较必须带 null** — 现有全部多模态对比实验都没做。

## 2. 方法核心(测量协议,非模型)

对每个 (数据集 d, 模态 m, 编码器 e) 三元组,冻结协议输出五元组:

```text
A(d,m,e) = ⟨ U_ds(流行度负例), U_ds(均匀负例), null 地板 μ/σ, 用户级 U 分布, 与流行度的相关 ⟩
```

- 数据集:Amazon 全类目(≥10)、ML1M/M3L、Steam、Yelp、MicroLens(短视频)、MIND(新闻)等 ≥20;
- 编码器:sentence-t5-xl / OpenAI-3 / SigLIP / CLIP / BLIP-2 caption→text;
- 产出:(a) 地图论文(哪些域的哪些模态落在"hard-negative 甜区 vs false-negative 险区");(b) `modality-preflight` pip 工具(输入 interactions+bank → 输出安全度报告);(c) 与已发表多模态方法收益的回溯相关分析 — **用地图解释文献中已知的"图像有时有害"散乱结论**。

## 3. 资产杠杆

U_ds 估计器、null 管线、raw-data-first 构建器直接横向扫描;φ 敏感性脚本变成地图的稳健性层。

## 4. 决定性实验

1. **回溯解释力**(核心赌注):收集 ≥8 篇已发表多模态方法在共享数据集上的每数据集增益,检验 U_ds 是否显著预测"内容信号收益/伤害"的方向 — 若解释力为零,地图只剩描述价值,降级为资源轨;
2. 编码器不变性:同数据集换编码器,U_ds 排序是否稳定(决定工具的实用性);
3. 抽样稳定性:4000 转移 × seed 方差 < 数据集间差的 1/10。

## 5. 风险与边界

- 已发表方法的每数据集数字抽取有对齐噪声(协议不一)→ 只做方向性秩检验,并在论文里明说;
- 纯测量论文在 RecSys 主轨的接受面窄 → 备选:资源轨(工具+地图数据集)或 SIGIR 短文。

## 6. 时间线

2026-07(AAAI 投稿后立即启动,纯 CPU)→ 2026-12 地图定稿 → 2027-04 RecSys 截稿。
