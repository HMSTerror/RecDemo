# Idea 04 — MLLM 证据审计员:离线审计、校准输出、服务路径外

- 目标会议:WSDM 2028(或 CIKM 2027)| 层级:C
- 一句话:MLLM 不当编码器也不当排序器,而是**离线审计每个物品的证据质量**(图文一致?描述可信?图像信息量?),输出经校准的证据分喂给任何门控/加权机制,再蒸馏成小校准器控成本。

## 1. Gap

1. MLLM4Rec 现有三种用法全在推理路径上:VLM 编码(VLM4Rec 2603.12625)、VLM agent 排序(ReasonRec 2606.28357,含不确定性委派)、MLLM 补内容(MMPCBench 2601.19750)— 共同问题:服务成本、幻觉直接进推荐决策;
2. 我们 AAAI'27 的 u/φ 是**统计 proxy**(一致性、AUC),测不了"这段描述是不是营销废话""图文是否矛盾"这类语义级证据缺陷 — 恰是 MLLM 强项;
3. 无人做"MLLM 判断 → 统计校准 → 蒸馏"的三段式:现有工作直接信 MLLM 原始输出(ReasonRec 的 uncertainty 是模型自报,无外部校准)。

## 2. 方法核心

```text
离线:MLLM 按冻结检查单对每物品打结构化标签(图文一致/描述具体度/图像可辨识度/夸大营销语)
校准:标签 → 与行为验证信号(train-only:退货率 proxy、复购、点击-转化落差)做等渗回归校准 → 证据分 q(i) ∈ [0,1]
蒸馏:小型双塔(冻结 embedding → q̂)拟合 q,新物品零 MLLM 调用
接入:q 作为锚点内部权重(c_h 中按 q 降权)或 ũ 的物品级因子 — 与门控框架即插即用
```

- 关键设计:MLLM 只在**离线批处理**出现,审计一次终身使用;检查单与校准映射预注册,防"prompt 调到结果好看"。

## 3. 决定性实验

1. **审计分的增量价值**(核心):在 field-coverage/长度等廉价 proxy 之上,q 对 (a) v1 每物品倾斜伤害、(b) 冷启动 CTR proxy 的偏相关是否显著非零 — 廉价 proxy 就够的话 idea 死;
2. 校准必要性:原始 MLLM 分 vs 等渗校准后,跨类目排序稳定性对比;
3. 蒸馏保真:q̂ 与 q 的秩相关 ≥ 0.85 且新类目不崩。

## 4. 风险

- MLLM 审计 10 万级物品的 API 成本(≈数百美元)— 用开源 Qwen-VL 级本地跑;
- "行为验证信号"里退货/转化在公开数据集不可得 → 用"高曝光低互动"落差 proxy,论文里如实声明弱监督;
- 若决定性实验 1 失败,唯一残值是审计数据集本身(可并入 Idea 02 的地图)。

## 5. 时间线

2027-01 决定性实验(依赖 02 的管线成熟)→ 2027-08 WSDM 截稿。
