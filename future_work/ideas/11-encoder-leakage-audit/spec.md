# Idea 11 — 基础编码器记忆泄漏:多模态推荐基准的完整性审计

- 目标会议:NeurIPS 2027 Datasets & Benchmarks | 层级:B
- 一句话:CLIP/SigLIP/T5 在网络级语料上预训练,大概率**见过 Amazon 商品页与 MovieLens 电影** — 多模态推荐的"内容信号收益"里有多少是编码器对共现结构的记忆而非泛化?设计探针量化,给主流基准发泄漏体检报告。

## 1. Gap

1. LLM 评测社区的数据污染审计已成显学,**推荐系统的多模态基准完全没做过**:所有 2024-2026 多模态推荐论文默认基础编码器是"中立特征器";
2. 我们的 U_ds 测量给了怀疑的具体形状:t5-xl 在 ML1M 上 text-AUC 0.754 — 电影标题+类型本不该有这么强的下一部预测力,**除非编码器吞过影评/共现语料**(IMDb、豆瓣式"看了又看");这从未被检验;
3. Binge Watch M3L(2602.15505)刚把 MovieLens 多模态化 — 新基准诞生之际正是立完整性标准的时机窗口。

## 2. 方法核心(三类探针,全部离线)

```text
时间探针:编码器 cutoff 之后上映/上架的物品子集 vs 之前 — U_ds 与推荐增益的落差即记忆上界估计
金丝雀探针:物品文本做保义改写(题名替换/实体匿名)重嵌入 — U_ds 塌多少 = 靠表面记忆多少
共现探针:embedding 相似度对"训练语料共现对"(影评同提及、同页商品)的判别力 vs 纯内容属性对
产出:主流基准 × 主流编码器的泄漏记分卡 + 去泄漏协议(改写清洗 / cutoff 切分)+ 修正后重排的方法榜
```

## 3. 决定性实验

1. **时间探针首查**(核心,2 周):M3L 电影按年代切,新片(>2023,晚于多数编码器 cutoff)U_ds 若显著低于老片(控制流行度/类型),泄漏实锤;若无差,idea 主线死 — 转为"多模态推荐对编码器记忆稳健"的短文(仍可发,但降级);
2. 金丝雀改写在 Amazon 描述上的 U_ds 敏感度;
3. 榜单扰动:≥3 个已发表方法在去泄漏协议下的排名变动幅度。

## 4. 风险

- "新片 U_ds 低"有混淆(新片行为数据也少)→ 需交互量配平;
- 指控性结论要求证据链极干净 — 预注册全部探针判据(§8.4 纪律模板),避免被读成 cherry-picking;
- 与 02 共享管线,即使主线死,产出并入地图论文。
